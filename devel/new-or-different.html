What’s New or Different¶

Warning

The Box-Muller method used to produce NumPy’s normals is no longer available in RandomGenerator. It is not possible to reproduce the random values using RandomGenerator for the normal distribution or any other distribution that relies on the normal such as the gamma or student’s t. If you require backward compatibility, a legacy generator, LegacyGenerator, has been created which can fully reproduce the sequence produced by NumPy.

  • random_entropy() provides access to the system source of randomness that is used in cryptographic applications (e.g., /dev/urandom on Unix).
  • Simulate from the complex normal distribution (complex_normal())
  • The normal, exponential and gamma generators use 256-step Ziggurat methods which are 2-10 times faster than NumPy’s default implementation in standard_normal(), standard_exponential() or standard_gamma().
  • The Box-Muller used to produce NumPy’s normals is no longer available.
  • All basic random generators functions to produce doubles, uint64s and uint32s via CTypes (ctypes()) and CFFI (cffi()). This allows these basic RNGs to be used in numba.
  • The basic random number generators can be used in downstream projects via Cython.

In [2]: import numpy.random

  • Optional dtype argument that accepts np.float32 or np.float64 to produce either single or double prevision uniform random variables for select distributions
    • Uniforms (random_sample() and rand())
    • Normals (standard_normal() and randn())
    • Standard Gammas (standard_gamma())
    • Standard Exponentials (standard_exponential())
In [7]: rg.seed(0)
Out[7]: RandomGenerator(Xoroshiro128) at 0x2B46BB0FB188

In [8]: rg.random_sample(3, dtype='d')
Out[8]: array([0.31483881, 0.00208035, 0.06399498])

In [9]: rg.seed(0)
Out[9]: RandomGenerator(Xoroshiro128) at 0x2B46BB0FB188

In [10]: rg.random_sample(3, dtype='f')
Out[10]: array([0.11265862, 0.31483877, 0.6936537 ], dtype=float32)

  • Optional out argument that allows existing arrays to be filled for select distributions

    • Uniforms (random_sample())
    • Normals (standard_normal())
    • Standard Gammas (standard_gamma())
    • Standard Exponentials (standard_exponential())

    This allows multithreading to fill large arrays in chunks using suitable PRNGs in parallel.

In [11]: existing = np.zeros(4)

In [12]: rg.random_sample(out=existing[:2])
Out[12]: array([0.06399498, 0.24822833])

In [13]: print(existing)
[0.06399498 0.24822833 0.         0.        ]
  • Support for Lemire’s method of generating uniform integers on an arbitrary interval by setting use_masked=True in (randint()).
In [14]: %timeit rg.randint(0, 1535, use_masked=False)
   ....: %timeit numpy.random.randint(0, 1535)
   ....:

1.44 us +- 6.21 ns per loop (mean +- std. dev. of 7 runs, 1000000 loops each)
